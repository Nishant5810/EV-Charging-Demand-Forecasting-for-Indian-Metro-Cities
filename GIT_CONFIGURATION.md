### ❌ Large Data Files (326.3 MB total)

These files are regenerated on every run, so they don't need to be in Git:

```
data/
├── raw_demand_data.csv               (67.9 MB)
│   └── 438,600 rows of weather + demand data
├── processed_demand_data.csv         (245.4 MB)
│   └── 434,400 rows of engineered features
└── predictions_test_set.csv          (13.0 MB)

**Why exclude?**

- Huge files increase clone time
- Can be regenerated by running `python run.py`
- Not needed for code collaboration
- GitHub has file size limits (100 MB max per file)

### ❌ Trained Models (56.5 MB total)

Pre-trained ML models that can be recreated:

```
models/
├── xgboost_model.json                (3.2 MB)
│   └── Primary forecasting model
├── prophet_Bengaluru_*.json          (5 files × 2.3 MB each)
├── prophet_Chennai_*.json            (5 files × 2.3 MB each)
├── prophet_Delhi_*.json              (5 files × 2.3 MB each)
├── prophet_Hyderabad_*.json          (5 files × 2.3 MB each)
└── prophet_Mumbai_*.json             (5 files × 2.3 MB each)

**Why exclude?**

- Binary/serialized format not suitable for version control
- Easily regenerated by running `python src/train.py`

- Not human-readable, hard to track changes

## ⚠️ Important Reminders

### DO NOT Commit These Files Manually

Even if they appear in `git status`, they should be ignored:

```bash
# ❌ NEVER do this
git add data/
git add models/
git add .env

# ✅ Git will automatically ignore them
git add .
```

### How to Regenerate Excluded Files

```bash
# Generate data + train models + launch dashboard
python run.py

# Or step-by-step
python src/data_ingestion.py        # Generate data
python src/feature_engineering.py   # Create features
python src/train.py                 # Train models
streamlit run src/dashboard.py      # Launch dashboard
```

### If You Accidentally Committed Files

```bash
# Remove from tracking (but keep locally)
git rm --cached data/*.csv
git rm --cached models/*.json

# Commit the removal
git commit -m "Remove large data/model files from tracking"

# Push to GitHub
git push
```
